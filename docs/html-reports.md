# HTML Reports

NowEDA can export a self-contained, dark-themed HTML report — no server required, no internet connection needed. Open it in any browser and share it with anyone.

---

## Generating a Report

### From Python

```python
from noweda.report.html import generate_html_report

report = df.noweda.report()
generate_html_report(report, "report.html")
```

### From the CLI

```bash
noweda data.csv --html report.html
```

---

## Report Sections

### Score Cards

Three prominent score cards at the top of the report:

| Card | What it shows |
|---|---|
| **Data Quality** | Score out of 100, colour-coded green/yellow/red |
| **Risk Level** | Risk score, colour-coded (low=green, high=red) |
| **Model Readiness** | Score out of 100, colour-coded |

---

### Insights

A dark-card list of every insight generated by NowEDA, ordered by category. Each insight is highlighted with a purple left-border accent.

---

### Column Schema Table

A table listing every column with:

- **Column name**
- **Dtype** (e.g., `int64`, `object`, `float64`)
- **Role** badge — colour-coded by type (purple=ID, cyan=categorical, green=numeric, lilac=datetime, grey=text)
- **Unique values** count
- **Uniqueness ratio** (unique / total rows)

---

### Missing Values

A table showing the missing rate for each column, with an inline colour-coded bar chart:

- Red bar — >30% missing
- Yellow bar — >0% missing
- Green bar — 0% missing

---

### Duplicates & Constants

A two-row summary table:

- Duplicate rows count and percentage
- List of constant (zero-variance) columns, or "None"

---

### Outliers

A table of numeric columns that have at least one outlier, sorted by outlier count descending.

---

### PII Detection (highlighted in red)

When PII is detected, this section appears with a **red border** to make it visually prominent. Lists each column and the findings.

---

### Encoding Detection (highlighted in yellow)

When encoded columns are detected, this section appears with a **yellow border**. Lists each column and the detected encoding type.

---

### Footer

Reports the NowEDA branding and the generation timestamp.

## Customising the Report

The HTML generator lives in [noweda/report/html.py](https://github.com/codewithdaniel1/NowEDA/blob/main/noweda/report/html.py). All styling is inline CSS — no external assets, making the file fully self-contained and portable.

To customise colours, add sections, or change the layout, modify `html.py` or subclass/replace `generate_html_report`.

### Example: add a custom section

```python
from noweda.report.html import generate_html_report as _base

def my_generate_html_report(report, output_path, custom_notes=""):
    # Generate the base report first (writes the file)
    _base(report, output_path)

    # Inject a custom section before </body>
    with open(output_path, "r") as f:
        html = f.read()

    custom_section = f"""
    <div class="section">
        <h2>Custom Notes</h2>
        <p>{custom_notes}</p>
    </div>
    """
    html = html.replace("</div>\n</body>", f"{custom_section}</div>\n</body>")

    with open(output_path, "w") as f:
        f.write(html)
```

## Sharing Reports

HTML reports are **fully self-contained** — all CSS is inline, no JavaScript, no external fonts. To share:

1. Send the `.html` file directly (email, Slack, etc.)
2. Upload to any static file host (S3, Google Drive, Dropbox)
3. Serve from any web server

The file will render identically in any modern browser without any dependencies.
