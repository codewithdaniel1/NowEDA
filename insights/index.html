    Insights Engine¶

    The Insights Engine translates raw plugin output and scores into plain-English, actionable statements. Instead of reading tables of numbers, you get a direct list of what matters and what to do about it.

    Getting Insights¶

    insights = df.noweda.insights()
# Returns: List[str]

for insight in insights:
    print(insight)

    Insight Categories¶

    Insights are generated in this order:

    1. Schema Insights¶

    Generated by the Schema plugin results.

    Trigger Insight
    One or more id_candidate columns "Likely identifier column(s) detected: [cols]. Consider excluding from modelling."
    One or more categorical or categorical_numeric columns "Column(s) with low cardinality (likely categorical): [cols]."
    One or more datetime columns "Datetime column(s) detected: [cols]. Temporal features may be valuable."

    2. Missing Value Insights¶

    Generated per column, based on missing rate.

    Missing rate Insight
    > 50% "Column '[col]' is X% missing — consider dropping it."
    30–50% "Column '[col]' has high missing rate (X%) — imputation recommended."
    0–30% "Minor missing values in: '[col1]', '[col2]', ..." (grouped)

    3. Duplicate and Constant Column Insights¶

    Trigger Insight
    Any duplicate rows "X duplicate row(s) detected (Y% of data). De-duplication is recommended."
    Any constant columns "Constant (zero-variance) column(s) detected: [cols]. These carry no information and can be dropped."

    4. Skewness Insights¶

    Generated from stats plugin results, per numeric column.

    Skewness Insight
    > 2 (right-skewed) "Column '[col]' is heavily right-skewed (skewness=X.XX). Log or power transform may help."
    < -2 (left-skewed) "Column '[col]' is heavily left-skewed (skewness=X.XX). Log or power transform may help."

    5. Outlier Insights¶

    Generated per numeric column with at least one outlier.

    Trigger Insight
    Any outliers "Column '[col]' has X outlier(s) (IQR method). Review before modelling."

    6. Correlation Insights¶

    Generated for each unique pair of numeric columns.

    Absolute correlation Insight
    > 0.9 "Very strong correlation (X.XX) between '[col1]' and '[col2]'. One may be redundant."
    0.7 – 0.9 "Strong correlation (X.XX) between '[col1]' and '[col2]'."

    Each pair is reported only once.

    7. PII Insights¶

    Generated per column with PII detected.

    Trigger Insight
    Any PII column "PII detected in column '[col]': X email address(es) found. Mask or remove before sharing."

    8. Encoding Insights¶

    Generated per column with encoding signals.

    Trigger Insight
    Any encoding detected "Column '[col]' may contain encoded data (possible_base64). Inspect for hidden payloads or obfuscation."

    9. Overall Score Insights¶

    Generated from the scores.

    Data quality:

    Score Insight
    ≥ 90 "Data quality score is excellent (≥90). Dataset looks clean."
    70 – 89 "Data quality score is acceptable (X). Minor issues present."
    < 70 "Data quality score is low (X). Significant cleaning required."

    Risk:

    Risk Insight
    0 "No risk signals detected."
    1 – 20 "Low risk level (X). Some sensitive indicators found."
    21 – 50 "Moderate risk level (X). Review PII and encoded columns before sharing."
    > 50 "High risk level (X). Sensitive data likely present — handle with care."

    Filtering Insights¶

    You can filter insights by keyword to focus on what matters for your use case:

    insights = df.noweda.insights()

# PII-related only
pii_insights = [i for i in insights if "PII" in i or "email" in i.lower()]

# Missing-data related only
missing_insights = [i for i in insights if "missing" in i.lower() or "dropping" in i.lower()]

# Correlation-related only
corr_insights = [i for i in insights if "correlation" in i.lower()]

# Risk-related only
risk_insights = [i for i in insights if "risk" in i.lower() or "sensitive" in i.lower()]

    Insights Source¶

    The insight generation logic lives in noweda/insights/generator.py. To add custom insight rules (e.g., business-domain checks), subclass InsightGenerator.
